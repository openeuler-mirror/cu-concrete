# cu-concrete 配置说明文档

## 一、云池配置（config.yaml）

### 1. 配置文件位置
`/opt/cu-concrete/api/config.yaml`

### 2. 云池配置格式

```yaml
{
    "pool-1": {
        "name": "云池1",
        "ansible_inventory": "/opt/cu-concrete/data/fetch/fetch.ini",
        "playbook_path": "/opt/cu-concrete/data/fetch/fetch.yml",
        "list": [
            '21','212','29','12'
         ]
    }
}
```

### 3. 新增云池步骤

要新增一个云池，需要在 config.yaml 中添加新的配置项：

```yaml
{
    "pool-1": {
        "name": "云池1",
        "ansible_inventory": "/opt/cu-concrete/data/fetch/fetch.ini",
        "playbook_path": "/opt/cu-concrete/data/fetch/fetch.yml",
        "list": [
            '21','212','29','12'
         ]
    },
    "pool-2": {                          # 新增云池ID（唯一标识）
        "name": "云池2",                  # 云池显示名称
        "ansible_inventory": "/opt/cu-concrete/data/fetch/pool2.ini",  # 该云池的Ansible主机清单文件
        "playbook_path": "/opt/cu-concrete/data/fetch/fetch.yml",      # 执行的Playbook路径（可复用）
        "list": [
            '21','212','29','12'           # 加固项列表（根据需求调整）
         ]
    }
}
```

### 4. 配置项说明

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `pool-1` | 云池唯一标识符（ID） | `pool-2`, `pool-prod` |
| `name` | 云池显示名称 | `生产环境云池` |
| `ansible_inventory` | Ansible 主机清单文件路径 | `/opt/cu-concrete/data/fetch/pool2.ini` |
| `playbook_path` | Ansible Playbook 文件路径 | `/opt/cu-concrete/data/fetch/fetch.yml` |
| `list` | 需要执行的加固项编号列表 | `['21','212','29','12']` |

### 5. 创建主机清单文件

每个云池需要对应一个 Ansible inventory 文件（可参考data/fetch/fetch.ini）：

```ini
# /opt/cu-concrete/data/fetch/pool2.ini
[all]
192.168.1.101
192.168.1.102 

[all:vars]
ansible_python_interpreter=/usr/bin/python3
```

## 二、加固脚本说明（fetch.yml）

### 1. 脚本位置
`/opt/cu-concrete/data/fetch/fetch.yml`

### 2. 脚本功能概述

该 Ansible Playbook 用于在远程主机上执行加固操作并收集结果。

### 3. 关键任务说明

| 任务名称 | 功能 | 说明 |
|----------|------|------|
| `Determine task identifier` | 确定任务标识 | 使用外部传入的任务ID或自动生成时间戳 |
| `Set backup root path` | 设置备份根路径 | 备份数据存储位置：`/opt/cu-concrete/data/backup/cu-concrete-{task_id}` |
| `Install cu-concrete package` | 安装 cu-concrete 包 | 使用 yum 安装（幂等操作） |
| `Preview hardening actions` | 预览加固操作 | 执行 `python3 main.py harden --list` 查看可加固项 |
| `Run hardening script` | **执行加固** | **关键任务：执行实际加固操作** |
| `Create and fetch tar.gz` | 打包并获取结果 | 将加固后的数据打包传回控制节点 |

### 4. 加固/还原配置修改

#### 4.1 执行加固（默认行为）

当前加固命令（第 65-70 行）：

```yaml
    - name: Run hardening script (python3 main.py harden --all)
      when: cu_dir_check.stat.exists
      ansible.builtin.command:
        cmd: python3 main.py harden items 2_1,2_12,2_9,1_2    # ← 修改这里
        chdir: "{{ cu_concrete_path }}"
      changed_when: true
```

**修改加固项**：
- 修改 `cmd` 中的 `items` 后面的编号列表
- 格式：`2_1,2_12,2_9,1_2`（对应 config.yaml 中的 list 配置）

#### 4.2 执行还原操作

如需改为**还原操作**，将上述任务改为：

```yaml
    - name: Run restore script
      when: cu_dir_check.stat.exists
      ansible.builtin.command:
        cmd: python3 main.py restore items 2_1,2_12,2_9,1_2    # ← harden 改为 restore
        chdir: "{{ cu_concrete_path }}"
      changed_when: true
```

**还原操作修改点**：
- 将 `harden` 改为 `restore`
- 还原项编号与加固时保持一致



## 三、快速操作指南

### 新增云池完整步骤

1. **创建主机清单文件**：
   ```bash
   vim /opt/cu-concrete/data/fetch/pool-new.ini
   ```

2. **编辑云池配置**：
   ```bash
   vim /opt/cu-concrete/api/config.yaml
   # 添加新的云池配置项
   ```

3. **（可选）创建专用 playbook**：
   ```bash
   cp /opt/cu-concrete/data/fetch/fetch.yml /opt/cu-concrete/data/fetch/pool-new.yml
   # 修改加固项或其他配置
   ```

4. **重启服务**（如果配置被缓存）：
   ```bash
   # 如果使用 uvicorn
   pkill -f uvicorn
   uvicorn config.asgi:application --host 0.0.0.0 --port 8000
   ```


## 四、注意事项

1. **云池 ID 唯一性**：每个云池的 ID 必须唯一，不能重复
2. **路径正确性**：确保 `ansible_inventory` 和 `playbook_path` 指向的文件存在
3. **主机清单权限**：确保 Ansible 可以连接到清单中的主机
4. **任务标识**：使用任务 ID 可以追踪一次加固操作的所有相关文件
